# SuperRight v1.0 蓝图（对齐超级右键）

> 目标：把 MVP 扩展成对齐「超级右键(iRightMouse)」的完整成品。
> 这是 v1.0 的设计与范围事实来源。基于超级右键功能清单（截图侧栏 + 官方功能列表）。

## 范围总览

沿用现有架构（FinderSync 沙盒扩展弹菜单 + `superright://` IPC + 非沙盒主程序执行 + App Group 共享配置），**不重构**，只新增动作、模块和一个设置 GUI。

### ✅ 已完成（MVP）
新建文件(文本类+自定义)、移动到/复制到、常用目录跳转、复制路径/名称、用 App 打开/在此打开、菜单栏图标。

### 🎯 v1.0 新增

**A. 新建文件增强**
- Office 空白模板：`docx` / `xlsx` / `pptx`（内置最小合法空文件，打包进 App 资源）+ `rtf`
- **模板目录**：扫描用户可配置的模板文件夹（默认 `~/Library/Application Support/SuperRight/Templates`），里面放什么就能右键新建什么——以此覆盖 Pages/Keynote/Numbers/WPS/AI/PSD 等任意类型，无需我逐一造模板
- 文本类、自定义名称（已）

**B. 文件操作增强**
- 剪切 / 拷贝 标记 + 「粘贴到此」（自维护剪贴板状态文件，粘贴时移动或复制）
- 永久删除（不进废纸篓，带确认弹窗）
- 移动到/复制到、复制路径/名称（已）

**C. 文件(夹)图标**
- 文件夹**换颜色**：取系统文件夹模板图标着色后 `NSWorkspace.setIcon`
- **设置自定义图标**（选图片文件）
- **拷贝图标 / 粘贴图标**（在文件间复制图标）
- **清除自定义图标**（恢复默认）

**D. 工具箱（收在「工具箱 ▸」子菜单）**
- 显示/隐藏 隐藏文件（toggle，`defaults write com.apple.finder AppleShowAllFiles` + 重启 Finder）
- 生成二维码（选中文件路径或剪贴板文本 → QR png 存到同目录/桌面）
- 压缩为 zip / 解压 zip
- 图片格式转换（png/jpg/heic/tiff/bmp/gif 互转，ImageIO）
- 文件信息（大小、创建/修改时间、权限、MD5、SHA256）
- 创建替身(alias) / 创建符号链接
- 批量重命名（前缀/后缀/起始序号/查找替换）

**E. 设置界面（SwiftUI 窗口，从菜单栏图标打开）**
- 通用：菜单栏图标开关、使用范围 scope、登录自启开关
- 列表管理：新建类型 / 常用目录 / 发送目标 / 打开 App 的增删改（告别手改 JSON）
- 功能开关：各菜单分组显示/隐藏
- 模板目录设置
- 权限引导：一键跳「完全磁盘访问」「Finder 扩展启用」
- 登录自启：`SMAppService`（macOS 13+）

### 🅿️ 明确不做（v1.1+，写出来避免误解）
- **翻译**（百度/谷歌）：需外部 API/网络密钥，偏离离线工具定位 → 延后
- **截图标注**：系统 `⌘⇧5` 已覆盖，自造代价大 → 不做
- Pages/Keynote/AI/PSD **内置**模板：用「模板目录」替代，用户自备

## 架构改动

### 配置 schema 扩展（向后兼容，缺字段用默认）
```jsonc
{
  "scope": "/",
  "newFileTypes": [...],            // 已有
  "favoriteDirs": [...],            // 已有
  "sendTargets": [...],             // 已有
  "openApps": [...],                // 已有
  "templatesDir": "~/Library/Application Support/SuperRight/Templates",
  "folderColors": [                 // 文件夹换色预设
    {"name":"红","hex":"#FF5A52"}, {"name":"橙","hex":"#FF9F0A"}, ...
  ],
  "features": {                     // 各分组显示开关，缺省全开
    "newFile": true, "transfer": true, "favorites": true,
    "icon": true, "toolbox": true, "openWith": true
  },
  "settings": { "menuBarIcon": true, "launchAtLogin": false }
}
```

### 新增 `superright://` 动作
`cut` / `copy` / `pasteHere`、`deletePermanently`、`toggleHidden`、`qrcode`、`zip` / `unzip`、`convertImage`(带 `fmt`)、`fileInfo`、`makeAlias` / `makeSymlink`、`batchRename`(带规则参数)、`setFolderColor`(带 `hex`) / `setCustomIcon` / `clearIcon` / `copyIcon` / `pasteIcon`、`newFromTemplate`(带模板路径)。

### 模块拆分（每个单一职责、可独立单测纯逻辑）
- 共享：`Config`(扩展字段)、`ActionURL`(新动作)、纯逻辑 helper（重命名规则、文件信息格式化、QR 文本、zip 命令拼接、图片格式枚举、folder color hex 解析）
- 主程序：
  - `FileOps`(已) + `TransferClipboard`(剪切/拷贝/粘贴状态)
  - `TrashOps`(永久删除)
  - `IconOps`(着色文件夹图标 / 自定义图标 / 拷贝粘贴 / 清除)
  - `Toolbox/`：`HiddenFiles`、`QRCode`、`Archive`、`ImageConvert`、`FileInfo`、`AliasOps`、`BatchRename`
  - `TemplateStore`(扫描模板目录) + 打包的 Office 模板资源
  - `Settings/`：SwiftUI 视图 + `SettingsWindowController`（AppKit 承载）+ `ConfigWriter`(GUI 改完写回 config.json)
  - `LoginItem`(SMAppService 自启)
- 扩展：`MenuBuilder` 扩展——新增「文件夹图标 ▸」「工具箱 ▸」分组，按 `features` 开关裁剪；高频项保持顶级，长尾收进子菜单

### 菜单结构（控制顶层不臃肿）
- 顶级：新建文件 ▸、复制路径、复制名称、移动到 ▸、复制到 ▸、用 App 打开 ▸、跳转到 ▸（空白处）
- 「文件夹图标 ▸」：换色子菜单 + 自定义/拷贝/粘贴/清除
- 「工具箱 ▸」：隐藏文件、二维码、压缩/解压、格式转换、文件信息、替身/软链、批量重命名、永久删除
- 「在此打开 ▸」「复制当前路径」「粘贴到此」（空白处）

## 测试策略（你只在最后统一测）
- **纯逻辑全部单测**（重命名规则、文件信息格式化、folder color 解析、zip 命令、QR 文本、图片格式映射、剪贴板状态序列化、配置新字段解析），降低"一次性测"的风险
- **app 侧动作用 `superright://` 自测脚本**：构建一个 `scripts/smoke.sh` 跑通所有可脚本化动作（新建各类型、zip、转图、二维码、文件信息、替身、重命名、永久删除、图标着色），输出可核对结果
- **需人工的**：Finder 菜单各项点击、设置 GUI 操作、隐藏文件 toggle 视觉确认——集中成一份「人工验收清单」给你一次过

## 风险/注意
- Office 模板需是合法最小 OOXML（zip 容器）；用打包的真实空白模板，不现造 XML
- 文件夹着色图标需生成像样的着色（非粗暴 tint）；先做"系统文件夹图标 + 色相叠加"，效果不满意再迭代
- 永久删除有数据风险 → 必须二次确认弹窗，且只删 Finder 传入的选中项
- 隐藏文件 toggle 是全局 Finder 设置，会重启 Finder（与超级右键一致）
- 设置 GUI 写回 config.json 与扩展读取的并发：GUI 写、扩展读，错开即可；写用原子替换
